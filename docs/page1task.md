# 페이지 1: 게시물 관리 세부 작업 명세서

## 📋 개요

본 문서는 게시물 관리(CRUD, 조회수, 상단고정) 기능 구현을 위한 세부 작업 내용을 정의합니다.

**우선순위**: P0 (필수)  
**총 기능 수**: 10개  
**총 테스트 수**: 15개

---

## 🔧 작업 목록

### 1. 데이터 모델 설계 및 구현

#### 1.1 게시물(Post) 엔티티 설계
- [ ] Post 엔티티 클래스 생성
  - `Id`: 고유 식별자 (long)
  - `Title`: 제목 (string, 필수, 최대 200자)
  - `Content`: 본문 (string, 필수)
  - `Category`: 카테고리 (string)
  - `Tags`: 태그 목록 (List<string>)
  - `AuthorId`: 작성자 ID (long)
  - `AuthorName`: 작성자명 (string)
  - `ViewCount`: 조회수 (int, 기본값 0)
  - `IsPinned`: 상단고정 여부 (bool, 기본값 false)
  - `IsDraft`: 임시저장 여부 (bool, 기본값 false)
  - `IsDeleted`: 삭제 여부 (bool, 기본값 false)
  - `CreatedAt`: 생성일시 (DateTime)
  - `UpdatedAt`: 수정일시 (DateTime)
  - `DeletedAt`: 삭제일시 (DateTime?)

#### 1.2 조회수 기록(ViewRecord) 엔티티 설계
- [ ] ViewRecord 엔티티 클래스 생성
  - `Id`: 고유 식별자 (long)
  - `PostId`: 게시물 ID (long)
  - `UserId`: 사용자 ID (long?, 비회원은 null)
  - `IpAddress`: IP 주소 (string, 비회원용)
  - `ViewedAt`: 조회 일시 (DateTime)

#### 1.3 데이터베이스 마이그레이션
- [ ] Posts 테이블 생성 마이그레이션 작성
- [ ] ViewRecords 테이블 생성 마이그레이션 작성
- [ ] 인덱스 생성 (AuthorId, Category, IsPinned, IsDeleted, CreatedAt)

---

### 2. API 엔드포인트 구현

#### 2.1 게시물 작성 (P1-001)
- [ ] POST `/api/posts` 엔드포인트 구현
- [ ] 요청 DTO 생성 (CreatePostRequest)
  - Title (필수)
  - Content (필수)
  - Category (선택)
  - Tags (선택)
- [ ] 응답 DTO 생성 (PostResponse)
- [ ] 유효성 검증 로직 구현
  - 제목 필수 검증
  - 제목 최대 길이 검증 (200자)
  - 본문 필수 검증
- [ ] 서비스 계층 로직 구현
- [ ] 단위 테스트 작성

#### 2.2 게시물 조회 (P1-002)
- [ ] GET `/api/posts/{id}` 엔드포인트 구현
- [ ] 게시물 상세 조회 로직 구현
- [ ] 삭제된 게시물 접근 방지 로직
- [ ] 조회수 증가 로직 연동
- [ ] 404 에러 처리 (게시물 미존재 시)
- [ ] 단위 테스트 작성

#### 2.3 게시물 수정 (P1-003)
- [ ] PUT `/api/posts/{id}` 엔드포인트 구현
- [ ] 요청 DTO 생성 (UpdatePostRequest)
  - Title (선택)
  - Content (선택)
  - Category (선택)
  - Tags (선택)
- [ ] 권한 검증 로직 구현 (작성자 또는 관리자만 수정 가능)
- [ ] 수정일시 자동 갱신 로직
- [ ] 403 에러 처리 (권한 없음)
- [ ] 단위 테스트 작성

#### 2.4 게시물 삭제 (P1-004)
- [ ] DELETE `/api/posts/{id}` 엔드포인트 구현
- [ ] 소프트 삭제 로직 구현 (IsDeleted = true, DeletedAt 설정)
- [ ] 권한 검증 로직 구현 (작성자 또는 관리자만 삭제 가능)
- [ ] 403 에러 처리 (권한 없음)
- [ ] 단위 테스트 작성

#### 2.5 게시물 목록 조회 (P1-005)
- [ ] GET `/api/posts` 엔드포인트 구현
- [ ] 쿼리 파라미터 처리
  - `page`: 페이지 번호 (기본값 1)
  - `pageSize`: 페이지 크기 (기본값 20, 최대 100)
  - `sortBy`: 정렬 기준 (createdAt, viewCount, title)
  - `sortOrder`: 정렬 순서 (asc, desc)
  - `category`: 카테고리 필터
  - `authorId`: 작성자 필터
- [ ] 페이지네이션 로직 구현
- [ ] 상단고정 게시물 우선 표시 로직
- [ ] 삭제된 게시물 제외 로직
- [ ] 페이지 응답 DTO 생성 (PagedResponse<PostSummaryResponse>)
- [ ] 단위 테스트 작성

#### 2.6 조회수 관리 (P1-006)
- [ ] 조회수 증가 서비스 메서드 구현
- [ ] 중복 조회 방지 로직 구현
  - 로그인 사용자: UserId 기준 24시간 내 중복 체크
  - 비로그인 사용자: IP 주소 기준 24시간 내 중복 체크
- [ ] ViewRecord 저장 로직
- [ ] 비동기 처리 고려 (성능 최적화)
- [ ] 단위 테스트 작성

#### 2.7 상단고정 설정 (P1-007)
- [ ] POST `/api/posts/{id}/pin` 엔드포인트 구현
- [ ] 관리자 권한 검증 로직
- [ ] IsPinned = true 설정 로직
- [ ] 403 에러 처리 (관리자 권한 없음)
- [ ] 단위 테스트 작성

#### 2.8 상단고정 해제 (P1-008)
- [ ] DELETE `/api/posts/{id}/pin` 엔드포인트 구현
- [ ] 관리자 권한 검증 로직
- [ ] IsPinned = false 설정 로직
- [ ] 403 에러 처리 (관리자 권한 없음)
- [ ] 단위 테스트 작성

#### 2.9 임시저장 (P1-009)
- [ ] POST `/api/posts/draft` 엔드포인트 구현
- [ ] 임시저장 게시물 생성 로직 (IsDraft = true)
- [ ] 기존 임시저장 덮어쓰기 옵션
- [ ] 단위 테스트 작성

#### 2.10 임시저장 목록 (P1-010)
- [ ] GET `/api/posts/draft` 엔드포인트 구현
- [ ] 현재 사용자의 임시저장 목록 조회 로직
- [ ] 페이지네이션 지원
- [ ] 단위 테스트 작성
